광고 계정 연결

네이버 검색광고 API 자격증명만 있으면 여러 계정을 한 대시보드에서 전환해 볼 수 있습니다.
1검색광고센터에서 API 라이선스 발급
searchad.naver.com 로그인 → 내 정보 → 도구 → API 사용 관리 → 라이선스 등록. 발급 즉시 API Key(Access License), Secret Key, Customer ID 3가지가 표시됩니다. Secret Key는 최초 1회만 전체 노출되니 즉시 안전한 곳에 저장하세요.
2accounts.json에 계정 정보 추가
함께 전달된 accounts.sample.json을 accounts.json으로 복사한 뒤, 아래처럼 계정을 배열로 나열합니다. 계정 추가는 배열에 객체 하나 더 붙이면 끝. 
[
  {
    "name": "선암파머스",
    "customerId": "1234567",
    "apiKey": "01000000000abc...",
    "secretKey": "AQAAAAA..."
  },
  {
    "name": "선암파머스 부계정",
    "customerId": "7654321",
    "apiKey": "...",
    "secretKey": "..."
  }
]
3데이터 수집 스크립트 실행
터미널에서 python fetch_naver_ads.py 실행. data/ 폴더에 계정별 JSON이 생성되고, 이 대시보드를 새로고침하면 상단 스위처에 자동으로 목록이 나타납니다.
4자동 갱신 (선택)
Cowork의 예약 작업(Scheduled Task) 기능 또는 OS 스케줄러(crontab / 작업 스케줄러)에 매일 새벽 4시 실행으로 등록하면, 매일 아침 최신 리포트가 준비됩니다.

🔗 광고 채널 연동 가이드

네이버 검색광고와 메타(페이스북·인스타그램)를 대시보드에 연결하는 방법이에요. (지금 보이는 네이버 값은 샘플 데이터이고, 연동되면 상단 배지가 "실시간 API"로 바뀝니다.)
🔍 네이버 검색광고
1API 라이선스 발급
searchad.naver.com 로그인 → 내 정보 → 도구 → API 사용 관리 → 라이선스 등록. API Key·Secret Key·Customer ID 3개를 확보하세요.
2accounts.json에 입력
함께 받은 accounts.sample.json을 accounts.json으로 복사 후 위 3개 값을 입력합니다. (여러 계정은 배열로 추가)
3수집 스크립트 실행
터미널에서 python fetch_naver_ads.py 실행 → data/ JSON과 bundle.js가 생성돼요. 대시보드를 새로고침하면 계정 스위처에 실계정과 "실시간 API" 배지가 떠요.
4자동 갱신 (선택)
예약작업·OS 스케줄러(또는 GitHub Actions)로 매일 새벽 스크립트를 돌리면 매일 아침 최신 데이터로 갱신됩니다.
※ 네이버 검색광고 API는 서버 인증(HMAC 서명)이 필요해 브라우저에서 직접 호출이 안 돼요. 그래서 스크립트로 받아 bundle.js를 갱신하는 방식이에요.
🔗 메타 (페이스북·인스타그램)
A지금 바로 — CSV 업로드
메타 광고 관리자 「광고」 탭에서 일별 CSV를 내보내 메타 탭에 업로드하면 즉시 반영돼요.
B자동 연동 — 액세스 토큰
개발자 토큰을 발급해 매일 자동 갱신하는 방식이에요. (개발자 계정 인증 필요)

메타(페이스북·인스타그램) 광고 계정 연결

메타는 액세스 토큰으로 연결합니다. 페이스북 페이지·광고 계정 권한이 담긴 토큰을 발급받아 등록하면, 네이버처럼 한 대시보드에서 메타 성과까지 볼 수 있습니다.
1메타 개발자 등록 & 앱 만들기
developers.facebook.com 에 평소 쓰는 페이스북 계정으로 로그인 → 우측 상단 내 앱(My Apps) → 앱 만들기 → 유형 비즈니스 선택 → 앱 이름(예: 선암파머스앱) 입력. 가입은 무료이고 새 계정이 아니라 기존 계정을 그대로 씁니다.
2Graph API 탐색기에서 토큰 발급
상단 도구 → Graph API 탐색기(developers.facebook.com/tools/explorer) → 우측에서 방금 만든 앱 선택 → Get Token → Get User Access Token → 아래 권한 체크 후 액세스 토큰 생성 → 페이지·광고계정 권한 허용. 화면에 뜨는 긴 문자열이 토큰입니다.
필요 권한: pages_show_list · pages_read_engagement · ads_read · read_insights
3장기 토큰으로 교환 (중요)
방금 받은 토큰은 몇 시간~최대 60일이면 만료됩니다. 아래 명령으로 60일짜리 장기 토큰으로 교환하세요. APP_ID/APP_SECRET은 앱 설정 → 기본 설정에서 확인합니다. 
curl "https://graph.facebook.com/v21.0/oauth/access_token?\
  grant_type=fb_exchange_token&\
  client_id=APP_ID&client_secret=APP_SECRET&\
  fb_exchange_token=단기토큰"
4accounts.json에 메타 자격증명 추가 & 자동 갱신
accounts.json의 계정 객체에 meta 블록을 추가합니다. adAccountId는 메타 광고관리자 URL의 act_ 번호, pageId는 페이지 정보에서 확인합니다. 
{
  "name": "선암파머스",
  "meta": {
    "accessToken": "EAAG...(장기토큰)",
    "adAccountId": "act_1234567890",
    "pageId": "111122223333"
  }
}
이후 python fetch_meta_ads.py 실행 → data/ 에 메타 성과 JSON 생성. 토큰이 60일마다 만료되니, 예약 작업으로 50일 주기 자동 갱신을 걸어두면 끊김 없이 유지됩니다.

📥 메타 광고 관리자 CSV 받는 법

캠페인별·소재별 성과를 보려면 광고 관리자에서 일별로 쪼갠 CSV를 받아 올리면 돼요. 캠페인용·소재용 두 가지만 기억하세요.
1기간 설정 & 일별로 쪼개기
우측 상단 달력에서 기간 선택 → 상단 분석 데이터(Breakdowns) → 시간 → 일별 체크. ⭐ 이게 있어야 날짜별 추세가 그려져요. (날짜는 "보고 시작" 열로 들어옵니다)
2열(컬럼) 맞춤설정에서 체크
상단 「열: 성과」 → 「열 맞춤 설정」 에서 아래 항목 체크 후 적용:
✅ 캠페인 이름  ·  ✅ 노출  ·  ✅ 링크 클릭  ·  ✅ 지출 금액  ·  ✅ 결과(또는 구매)  ·  ✅ 구매 전환값
⭐ 「광고」 탭에서 받을 땐 "캠페인 이름"을 꼭 추가하세요. 그래야 한 파일로 캠페인별·소재별이 모두 채워져요. (광고 이름은 광고 탭이면 자동 포함)
3「광고」 탭에서 CSV 내보내기 — 파일 하나면 끝
좌측에서 「광고」 탭을 선택한 뒤 우측 상단 내보내기(↓) → 테이블 데이터 내보내기 → CSV. 광고 단위로 받으면 광고 이름 + 캠페인 이름이 함께 들어와서, 메타 탭에 한 번만 올리면 캠페인별·소재별 표가 동시에 채워져요.
※ 「캠페인」 탭에서 받아도 동작하지만, 그건 캠페인별 표만 나오고 소재별은 안 나와요.
⚠️ 꼭 지켜요
· 반드시 CSV로 받기 (엑셀 .xlsx는 안 읽혀요 — 받았으면 "다른 이름으로 저장 → CSV"로 변환)
· 2단계 "일별" 빼먹으면 "일자 컬럼을 찾을 수 없습니다" 에러가 나요

리포트 자체검증 · —
모든 수치는 데이터 소스의 합계·평균으로 재계산된 값입니다. API 연동 후에는 매일 새벽 스크립트가 원본 로그를 갱신하며, 본 검증 블록이 불일치 항목을 자동 표시합니다.
※ 업계 참고치는 농식품·신선식품 이커머스 통상범위(쇼핑검색 CTR 1.0~1.5%, CPC 200~400원, ROAS 300~500%)를 기본값으로 사용했으며, 실 운영 시 네이버 검색광고센터의 동종업계 비교 탭 수치를 주 1회 수동 반영 권장합니다.
